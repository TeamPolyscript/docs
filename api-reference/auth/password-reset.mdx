---
title: "Password recovery / reset"
description: "Polyscript forgot-password flow: request a recovery email, verify the token from the magic link, and set a new password for the account."
---

## Request reset

```http
POST /auth/password/recovery
Content-Type: application/json
```

```json
{ "email": "you@example.com" }
```

Always returns `200 { ok: true }` regardless of whether the email exists — prevents account enumeration.

If the email matches a user, a 43-char URL-safe token is generated, stored in `passwordResets` collection (TTL 30 min), and emailed as a link:

```
https://polyscript.io/reset-password?token=yh3jK_8s2xQv9pL-mNoP4qR6tU8wX0zA-bC2dE4fG6hI
```

### Errors

- `400 invalid_email` — format only (no enumeration)
- `429` — more than 3 / hour

---

## Peek (validate token)

```http
GET /auth/password/reset?token=<token>
```

Read-only check — does **not** consume the token. The frontend uses this to decide whether to render the "set new password" form or the "expired link" message.

```json
{ "ok": true }
```

### Errors

- `400 invalid_or_expired`

---

## Consume (set new password)

```http
POST /auth/password/reset
Content-Type: application/json
```

```json
{
  "token": "yh3jK_8s2xQv...",
  "password": "new_password_min_8"
}
```

<ParamField body="token" type="string" required />
<ParamField body="password" type="string" required>Minimum 8 characters.</ParamField>

Atomic operation: marks token `consumedAt = now()` and updates `users.password.hash` in the same transaction. Token is single-use.

```json
{ "ok": true }
```

### Errors

- `400 invalid_or_expired` — token unknown, already consumed, or past `expires_at`
- `400 password_too_short`

### After reset

User is redirected to `/login` to sign in with the new password. The reset doesn't auto-log them in.
